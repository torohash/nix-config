# エージェントオーケストレーション

## 利用可能なエージェント

`~/.claude/agents/` に配置:

| エージェント | 目的 | 使用タイミング |
|-------------|------|---------------|
| planner | 実装計画の作成 | 複雑な機能実装、アーキテクチャ変更、リファクタリング |
| code-reviewer | コードレビュー | コードの作成・変更後 |
| tdd-guide | テスト駆動開発 | 新機能の実装、バグ修正、リファクタリング |
| explore | コードベース探索 | 複数モジュールにまたがる構造把握、不慣れなコードベースの調査 |
| librarian | 外部ライブラリ調査 | 不慣れなライブラリの使い方、公式ドキュメントの検索、OSS の実装例の発見 |

## エージェントの即時使用

ユーザーからの明示的な指示がなくても、以下の場面で積極的に使用する:

1. 複雑な機能リクエスト → **planner** エージェント
2. コードの作成・変更後 → **code-reviewer** エージェント
3. バグ修正・新機能の実装 → **tdd-guide** エージェント
4. 不慣れなライブラリの調査 → **librarian** エージェント
5. コードベースの構造把握 → **explore** エージェント

## 並列タスク実行

独立した操作には常に並列の Task 実行を使用する:

```markdown
# GOOD: 並列実行
3つのエージェントを並列で起動:
1. エージェント 1: 認証モジュールのコードレビュー
2. エージェント 2: キャッシュシステムの実装調査
3. エージェント 3: ユーティリティの型チェック

# BAD: 不要な逐次実行
まずエージェント 1、次にエージェント 2、次にエージェント 3
```
