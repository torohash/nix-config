## 調査・検索の委譲ルール（確率的）

- **code の調査 / web search / code search は、原則 Codex に依頼する。**
  - これは確率的なルール（必ずではない）。**適宜判断**でよい。Codex が使えない・明らかに即答できる・1コマンドで済む等なら自分でやってよい。
- **切り替えトリガー（重要）**: 自分で始めた調査でも、(a) 2手以上の横断調査になった、(b) 1回でも大きな出力（数十KB級・ログ/grep の大量ヒット）が返った、(c) 次の一手が「あと1コマンドだけ」を繰り返し始めた、のいずれかに当てはまった時点で「単発 lookup」ではなく「多段調査」と再判定し、**そこから先は Codex に委譲し直す**。惰性で inline 調査を続けない。
- 目的: Claude の context を調査結果や検索ノイズで埋めない。重い読込・横断検索・web 調査は Codex に逃がし、Claude は **方針決定・監査・ユーザー対話**に集中する。
- 例:
  - 「X はどこに実装されている？」「Y の使い方」「Z をネットで調べて」→ Codex に依頼。
  - 大量ファイルの横断 grep / 仕様の追跡 / ライブラリ内部の調査 → Codex に依頼。
- Codex の戻り（調査結果・数値・出典）は **verbatim で素通ししない**。Claude が監査・照合してから確定する（レビュー・監査ルールに従う）。
