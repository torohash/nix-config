# コード変更を委譲する

メインエージェントは要求整理、担当選定、差分確認、検証、完了報告を担当する。実装は原則として選択した単一のサブエージェントへ委譲する。

## 選定して起動する

1. ユーザー要求、作業対象へ適用される`AGENTS.md`、既存変更、変更範囲、成果が保証すべき振る舞い、境界条件、許容する仮定、意図的な対象外、完了条件を確認する。
2. 変更量ではなく、必要な判断と設計責任から担当種別を選ぶ。
3. 依存関係、曖昧さ、危険性、検証負荷から推論レベルを選ぶ。最低でも`medium`を使い、Solでは最低`high`を使う。
4. 起動時に`agent_type`、`model`、`reasoning_effort`をすべて明示する。次の対応と異なるモデルを指定しない。
5. 目的、変更範囲、成果が保証すべき振る舞い、境界条件、許容する仮定、意図的な対象外、制約、完了条件、検証、変更禁止範囲を渡し、書き込み担当を1体だけ起動する。
6. 差分と検証結果を確認し、メインエージェントが必要な最終検証と完了報告を行う。

| 担当種別 | 起動時のモデル | 選択条件 |
|---|---|---|
| `code-change-mechanical` | `gpt-5.6-luna` | 仕様、変更内容、実装経路、完了条件がすべて明確で、設計判断が不要なコメント追加、指示済みの定数化、厳密な名前変更、定型変換などを任せる。 |
| `code-change-standard` | `gpt-5.6-terra` | 通常の機能追加、バグ修正、テスト追加、既存設計内のリファクタリングなど、既存方針内の限定的な実装判断を任せる。 |
| `code-change-advanced` | `gpt-5.6-sol` | 新たな設計選択、アーキテクチャ変更、大規模な再構成、移行、複数システム間の変更、高リスク領域を任せる。 |

変更量が多くても機械的なら`code-change-mechanical`を選べる。変更量が少なくても設計判断や高い危険性があれば`code-change-standard`または`code-change-advanced`を選ぶ。

## 推論レベルを選ぶ

| レベル | 選択条件 |
|---|---|
| `medium` | 実装経路がほぼ一意で、影響範囲と検証方法が明確である。 |
| `high` | 複数の実装候補、非自明な制御・データフロー、互換性、境界条件、原因調査がある。 |
| `xhigh` | 複数コンポーネントの相互作用、設計上のトレードオフ、広い影響範囲、非同期処理、移行、検証上の不確実性がある。 |
| `max` | 分割しにくい最難関問題で、失敗時の影響または不可逆性が大きい、あるいは過去の修正が繰り返し失敗している。Solだけで使う。 |

担当別の推論レベルは次の範囲に限定する。

- Luna: `medium`、`high`
- Terra: `medium`、`high`、`xhigh`
- Sol: `high`、`xhigh`、`max`

## 委譲契約を渡す

指示に次の項目を具体的に記載する。

```text
目的:
変更範囲:
成果が保証すべき振る舞い:
境界条件:
許容する仮定:
意図的な対象外:
守るべき制約:
完了条件:
実行する検証:
変更してはいけないもの:
担当種別または推論レベルを変更すべき条件:
```

外部から見える成果の意味、正しさ、適用範囲を変える判断を、実装担当に暗黙に補わせない。これらを左右する仕様が確定していない場合は、通常の内部実装の選択と区別してメインエージェントへ返させる。一方、契約済みの振る舞いを変えない定型的な内部実装の選択まで、ユーザー判断として差し戻させない。

同じファイルまたは同じ責任範囲を複数の書き込み担当へ委譲しない。独立した調査、検証、レビューだけを並列化できるが、書き込み担当は常に1体に保つ。

## 未完了時に再選定する

未完了の理由を、推論不足、担当範囲の不一致、情報・権限・ユーザー判断の不足に分ける。

- 推論不足なら、既存スレッドへ追加指示して続行させない。目的、必要な作業文脈、前回の失敗理由、実施済み作業、未実施作業を含め、同じ担当種別を1段階高い推論レベルで新規起動する。
- Lunaは`medium`から`high`へ上げる。`high`でも不足するなら作業の性質を再評価し、`code-change-standard`をTerraで起動する。
- Terraは`medium`から`high`、次に`xhigh`へ上げる。`xhigh`でも不足するか設計責任が発生したら、`code-change-advanced`をSolで起動する。
- Solは`high`から`xhigh`、次に`max`へ上げる。
- 機械的変更の範囲を超えたら`code-change-standard`へ、既存設計内の判断を超えたら`code-change-advanced`へ変更する。役割境界を超えた場合は、同じ担当種別の推論レベルだけを上げない。
- 不足していた指示や作業中に得られた事実を補うだけなら、同じスレッドへ追加指示できる。情報、権限、外部状態またはユーザー判断が不足する場合は、推論レベルを上げず不足を解消する。

新規起動前に前の担当を終了させ、書き込み担当が同時に複数動かないようにする。
