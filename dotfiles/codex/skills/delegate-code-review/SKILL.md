# コードレビューを委譲する

実装担当とは別の`code-review`を、必ずSolかつ読み取り専用で起動する。レビュー担当に修正を実装させない。

## 推論レベルを選ぶ

| レベル | 選択条件 |
|---|---|
| `high` | 通常の機能追加、バグ修正、テスト追加、限定的な再構成の差分を確認する。 |
| `xhigh` | 複数コンポーネント、アーキテクチャ変更、移行、セキュリティ、認証、並行処理、データ整合性、広い回帰範囲を確認する。 |
| `max` | 分割しにくい最難関の差分、重大な不可逆性がある変更、または過去のレビューで問題を繰り返し見落としている変更を確認する。 |

## 起動して結果を受け取る

1. レビュー対象の差分、要求、適用される`AGENTS.md`、実施済みの検証、既知の制約を整理する。
2. `agent_type`に`code-review`、`model`に`gpt-5.6-sol`、`reasoning_effort`に選んだ`high`、`xhigh`または`max`を明示して新規起動する。
3. 読み取り専用で作業し、編集、コミット、追加のサブエージェント起動を行わないよう明記する。
4. 問題を重大度順に報告させ、それぞれに根拠、ファイルと行などの場所、具体的な影響、修正方向を対応付けさせる。
5. 問題がない場合はその旨と、検証していない範囲または残る危険を返させる。
6. メインエージェントが指摘を確認し、修正が必要なら`$delegate-code-changes`を使って実装担当へ別途委譲する。

## 未完了時に再起動する

推論不足なら、既存スレッドへ追加指示して続行させない。レビュー対象、必要な文脈、前回の失敗理由、確認済み範囲、未確認範囲を含め、同じ`code-review`を`high`から`xhigh`、次に`max`へ1段階ずつ上げて新規起動する。

差分、仕様、検証結果、再現条件などの情報が不足する場合は推論レベルを上げず、不足を解消する。不足していた情報を補うだけなら同じスレッドへ追加指示できる。
