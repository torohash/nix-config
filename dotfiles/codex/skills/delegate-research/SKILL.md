# 調査を委譲する

調査対象をプロジェクト内またはWebに分け、質問に答えるために必要な範囲だけを読み取り専用担当へ委譲する。

## 選定して起動する

1. 調査によって決めたいこと、必要な根拠、最新性の要否、調査禁止範囲を明確にする。
2. ローカルのコード、設定、テスト、文書を根拠にするならプロジェクト内調査を選ぶ。公開情報、現行仕様、更新状況を根拠にするならWeb調査を選ぶ。両方が必要なら責任範囲を分ける。
3. 対象の明確さ、横断範囲、資料間の矛盾、誤りの影響から担当種別と推論レベルを選ぶ。
4. 起動時に`agent_type`、`model`、`reasoning_effort`をすべて明示する。次の対応と異なるモデルを指定しない。
5. 読み取り専用、編集・コミット・追加のサブエージェント起動禁止を明記して起動する。

| 担当種別 | 起動時のモデル | 選択条件 |
|---|---|---|
| `project-research-lookup` | `gpt-5.6-luna` | 対象ファイル、識別子、設定項目または質問が明確で、限定されたコードや文書から位置と根拠を特定する。 |
| `project-research-synthesis` | `gpt-5.6-terra` | 複数のファイル、設定、テスト、文書または処理経路を横断し、関係、原因候補、影響範囲を統合する。 |
| `project-research-deep` | `gpt-5.6-sol` | 全体設計の理解、難しい原因調査、複数コンポーネントに及ぶ広範な影響分析を行う。 |
| `web-research-lookup` | `gpt-5.6-luna` | 調べる事実と参照すべき公式サイト、仕様書、公開資料がほぼ明確で、最新情報と根拠URLを確認する。 |
| `web-research-synthesis` | `gpt-5.6-terra` | 複数の公式資料や一次資料を比較し、公開日、更新日、対象範囲、記述の矛盾を確認して統合する。 |
| `web-research-deep` | `gpt-5.6-sol` | 不完全・対立する資料、専門性と探索範囲の広い難しい調査、誤りの影響が大きい判断を扱う。 |

## 推論レベルを選ぶ

最低でも`medium`を使い、Solでは最低`high`を使う。

| レベル | 選択条件 |
|---|---|
| `medium` | 対象と質問が明確で、調査経路と十分な根拠の条件がほぼ一意である。 |
| `high` | 複数の処理経路や資料、非自明な関係、更新日の確認、原因候補または軽微な矛盾がある。 |
| `xhigh` | 全体設計の再構成、広い影響分析、重大な矛盾、専門性が高い資料評価、検証上の不確実性がある。 |
| `max` | 分割しにくい最難関調査で、誤った結論の影響が大きい、または過去の調査が繰り返し結論へ到達できていない。Solだけで使う。 |

担当別の推論レベルは、Lunaを`medium`、`high`、Terraを`medium`、`high`、`xhigh`、Solを`high`、`xhigh`、`max`に限定する。

## 調査契約を渡す

プロジェクト内調査には、結論、根拠となるファイル位置と該当内容、確認済み範囲、未確認範囲を返させる。確定事項と推定を区別させる。

Web調査には、依頼時点の最新性を確認させ、可能な限り一次資料を優先させる。結論、事実を直接支える根拠URL、公開日または更新日、資料の適用範囲、残る矛盾を返させる。

独立していて結論の重複しない調査だけを並列化する。異なる担当へ同じ事実を重複して調べさせず、調査結果の統合責任はメインエージェントが持つ。

## 未完了時に再選定する

- 推論不足なら、既存スレッドへ追加指示して続行させない。質問、必要な作業文脈、前回の失敗理由、確認済み範囲、未確認範囲を含め、同じ担当種別を1段階高い推論レベルで新規起動する。
- Lunaは`medium`から`high`へ上げる。`high`でも不足するなら、同じ調査領域の`synthesis`担当をTerraで起動する。
- Terraは`medium`から`high`、次に`xhigh`へ上げる。`xhigh`でも不足するか全体設計・高リスク判断の責任が発生したら、同じ調査領域の`deep`担当をSolで起動する。
- Solは`high`から`xhigh`、次に`max`へ上げる。
- 限定検索から複数根拠の統合が必要になったら`lookup`から`synthesis`へ、全体設計、難しい原因、高リスク判断が必要になったら`deep`へ変更する。役割境界を超えた場合は、同じ担当種別の推論レベルだけを上げない。
- 不足していた質問や探索対象を補うだけなら、同じスレッドへ追加指示できる。情報、権限、外部状態またはユーザー判断が不足する場合は、推論レベルを上げず不足を解消する。
